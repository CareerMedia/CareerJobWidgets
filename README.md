# Career Job Widgets (Static, GitHub Pages)

A modern, responsive static web app (Vite + React + TypeScript) that lets an admin curate multiple RSS/JSON job feeds and publish **embeddable widgets** (iframe-based) for other websites.

## Key constraints (by design)

- Runs **entirely in the browser** (GitHub Pages / static hosting).
- **No backend**, no database.
- Admin portal “protection” is a **lightweight shared-password gate** (client-side hash + `sessionStorage`). This is *not enterprise-grade security* because the JS bundle is public on a static site.
- Feed configs persist in **`localStorage`**.
- Feeds may fail due to **CORS**. If a source doesn’t allow browser fetching from your Pages origin, this static app cannot bypass it without a proxy (intentionally not used).

## Deploy to GitHub Pages

This app uses `HashRouter`, so routes work on GitHub Pages without server rewrites.

### Important: deploy the **built** `dist/` folder

Do **not** publish the repo root as-is. The source `index.html` references `/src/main.tsx`, which only works in Vite dev — it will 404 on GitHub Pages and show a blank white page.

### Recommended: GitHub Actions (included)

1. In GitHub: **Settings → Pages → Build and deployment → Source** → choose **GitHub Actions**.
2. Push to `main`. The workflow `.github/workflows/deploy-pages.yml` builds `dist/` and deploys it.
3. Live URL: `https://careermedia.github.io/CareerJobWidgets/`

Production builds use base path `"/CareerJobWidgets/"` (see `vite.config.ts`).

## Admin portal

- Route: `/#/admin`
- Login: shared password gate (hash check + `sessionStorage`)

### Change the shared admin password hash

1. Pick a new password.
2. Compute `sha256(password)` as base64 using the helper in `src/services/security/passwordGate.ts`.
3. Paste it into `src/config/appConfig.ts` as `ADMIN_PASSWORD_SHA256_BASE64`.

File: `src/config/appConfig.ts`

- `ADMIN_PASSWORD_SHA256_BASE64`
- `ADMIN_SESSION_KEY`

## Feed configuration

In the admin portal you can:

- Add / edit / delete feeds
- Enable/disable feeds
- Test a feed (browser fetch + parse)
- Export JSON (download)
- Import JSON (upload)

Supported feed types:

- **RSS**: RSS 2.0 XML parsed with `DOMParser` (`item.link` is used as the apply URL)
- **JSON**: flexible extraction from common shapes (array or `{ jobs/items/results/data: [] }`)

## Embeds (critical requirement)

All widgets are embed-first and use **iframe embeds** for strong style isolation:

- The widget’s CSS won’t leak to the host page.
- The host page’s CSS won’t break the widget (within the iframe).

Embed routes render **only the widget** (no app chrome):

- `/#/embed/all-jobs`
- `/#/embed/featured-jobs`
- `/#/embed/feed-tabs`
- `/#/embed/feed/:feedId/directory`
- `/#/embed/feed/:feedId/featured`

### Embed code generation

Embed codes are generated by `src/services/embeds/embedCode.ts` using:

- production base URL (optional) in `src/config/appConfig.ts` (`PRODUCTION_PAGES_URL`)
- fallback to `window.location.origin + window.location.pathname`

### Example iframe snippet

```html
<iframe
  src="https://YOUR_GITHUB_PAGES_URL/#/embed/all-jobs"
  title="All jobs directory"
  width="100%"
  height="760"
  style="border:0; width:100%; max-width:100%;"
  loading="lazy"
></iframe>
```

## CORS troubleshooting (Handshake and similar feeds)

Some feeds — **including Handshake RSS** — work in a browser tab but **fail when fetched from JavaScript** because the server does not send `Access-Control-Allow-Origin` headers.

This app works around that with a **server-side feed sync** stored in the repo:

1. Add CORS-blocked feeds to `feeds.sync.json`.
2. Run the **Sync job feeds** GitHub Action (daily at 6 AM Pacific, or manually from Admin → **Sync feeds now**).
3. The action fetches feeds server-side and commits JSON to `public/data/feeds/`.
4. The site loads that same-origin cache automatically — no browser CORS needed.

### Add a Handshake feed

1. Add an entry to `feeds.sync.json` with the feed URL and a stable `id`.
2. Click **Sync feeds now** in the admin portal (or run the GitHub Action manually).
3. After the workflow commits and deploys, jobs appear on the site.

Optional: store the feed URL in a GitHub secret (`HANDSHAKE_CSUN_AMC_FEED_URL`) and set `urlEnv` in `feeds.sync.json`.
